# BotForge V0.01 (RU)

**Визуальный конструктор автоматизации для игр и программ без единой строчки кода.**

Вы собираете бота из готовых блоков: «если на экране появилось это кликни сюда», «жди, пока не прозвучит звук», «зажми клавишу на 2 секунды». BotForge сам следит за экраном, слушает звук и выполняет действия по сценарию.

Работает на Windows.

## 🧠 Как это работает

Идея проста бот состоит из трёх вещей:

```
ТРИГГЕРЫ (что искать)  →  СЦЕНАРИЙ (блоки по порядку)  →  ДЕЙСТВИЯ (что делать)
```

1. **Триггер** это «глаз» бота. Он смотрит на свою область экрана (или слушает звук) и отвечает на один вопрос: *сработал или нет?* Например: «в этом углу появилась красная полоска HP» или «в игре прозвучал звук поклёвки».

2. **Сценарий** список блоков, которые выполняются сверху вниз. Блоки бывают простые («сделай действие», «подожди») и условные («ЕСЛИ триггер сработал сделай это», «ПОКА триггер активен повторяй это», «ЖДИ, пока триггер не сработает»).

3. **Действие** это «руки» бота: клики, нажатия клавиш, ввод текста, перетаскивания, паузы.

Когда вы жмёте ▶, движок начинает крутить сценарий в цикле: проходит все блоки по порядку, доходит до конца начинает заново (если включён цикл). При этом в интерфейсе видно, какой шаг выполняется прямо сейчас, а триггеры подсвечиваются, когда срабатывают.

**Важно:** каждый триггер сканирует **только свою выбранную область экрана**, а не весь экран. Это быстрее и точнее выделяйте область как можно меньше.

## 🚀 Быстрый старт

Нужен **Python** и **Windows**.

```bash
# 1. Клонировать репозиторий
git clone https://github.com/Decursusss/BotForge.git
cd BotForge

# 2. Создать виртуальное окружение
python -m venv .venv
.venv\Scripts\activate

# 3. Установить зависимости
pip install -r requirements.txt

# 4. Запустить
python main.py
```

Откроется главное окно с тремя панелями:

```
┌─────────────┬──────────────────┬─────────────────────┐
│  ТРИГГЕРЫ   │     СЦЕНАРИЙ     │  ПРЕВЬЮ + НАСТРОЙКИ │
│             │                  │                     │
│ список      │ блоки сверху     │ живое видео области │
│ триггеров   │ вниз, drag&drop  │ + редактор свойств  │
│             │                  │   выбранного блока  │
└─────────────┴──────────────────┴─────────────────────┘
```

### Триггеры

| Тип | Значок | Что делает | Когда использовать |
|---|---|---|---|
| **HSV-маска** | `H` | Ищет цвет в области (диапазон оттенков). Есть пипетка кликни по превью и цвет подберётся сам | Полоски HP/маны, цветные индикаторы, кнопки |
| **Картинка** | `T` | Ищет картинку-образец на экране (template matching). Образец вырезается прямо в приложении | Кнопки, иконки, любые элементы интерфейса |
| **Пиксель** | `P` | Следит за цветом одного пикселя | Самый быстрый способ: «пиксель сменил цвет = что-то произошло» |
| **Изменение** | `Δ` | Замечает любое движение/изменение в области | «Что-то появилось/зашевелилось», когда не важно что именно |
| **Звук** | `S` | Слушает звук системы. Два режима: совпадение с записанным образцом или просто громкость | Звук поклёвки в рыбалке, сигнал тревоги, любые игровые звуки |

У каждого триггера есть **кнопка «Проверить»** мгновенный тест без запуска бота, с зелёным маркером места находки на превью.

Звуковой триггер можно **записать прямо в программе** (она запишет системный звук), обрезать тишину и прослушать.

### Блоки сценария логика бота

| Блок | Значок | Что делает |
|---|---|---|
| **Действие** | ⚡ | Просто выполняет одно действие |
| **Ожидание** | ⏱ | Пауза N миллисекунд |
| **ЕСЛИ триггер** | ❓ | Триггер сработал → выполнить **цепочку действий** (можно несколько подряд) |
| **Ждать триггер** | ⏳ | Стоит и ждёт, пока триггер не сработает (с таймаутом или бесконечно) |
| **ПОКА триггер** | 🔁 | Повторяет цепочку действий, пока триггер активен (или наоборот — пока НЕ активен, есть инверсия) |

Блоки можно **перетаскивать** мышкой, **дублировать**, **удалять** (кнопка ✕, правый клик или Del).

### Прочее

- ▶ **Живая визуализация**: подсветка текущего шага, счётчик «Итерация N · Шаг i/k», лог событий
- 🖼 **Живое превью** любой области экрана с наложением маски цвета и маркерами находок
- 💾 **Сохранение/загрузка** проектов в обычный JSON (старые проекты открываются формат обратно-совместимый)
- 🛑 **F12 аварийный стоп** из любого места, даже когда игра в фокусе
- 🎮 **Два режима ввода**: `pydirectinput` (видят игры с DirectInput) и `pyautogui` (обычные программы)
- 🔄 Цикл сценария: бесконечно или N итераций, с настраиваемой задержкой

## 📁 Структура проекта

```
BotCrafter/
├── main.py                      # точка входа (+ DPI-awareness для Windows)
├── requirements.txt
└── botforge/
    ├── core/                    # логика, ничего про интерфейс не знает
    │   ├── models.py            # данные: Trigger, Action, Block, BotConfig
    │   ├── engine.py            # BotEngine (QThread) — цикл исполнения сценария
    │   ├── capture.py           # захват экрана (mss)
    │   ├── vision.py            # компьютерное зрение: HSV-маска, поиск картинки,
    │   │                        #   пиксель, детектор изменений (OpenCV)
    │   ├── audio.py             # звук: запись системного звука (loopback),
    │   │                        #   спектрограммы, сравнение с образцом
    │   ├── input_backend.py     # ввод: pydirectinput / pyautogui + WinAPI-фоллбэк
    │   ├── recorder.py          # запись действий пользователя (pynput)
    │   └── storage.py           # сохранение/загрузка проекта в JSON
    └── ui/                      # интерфейс (PySide6)
        ├── main_window.py       # главное окно, связывает всё вместе
        ├── preview_widget.py    # живое превью области + маркеры находок
        ├── region_overlay.py    # полупрозрачный оверлей выбора области/точки
        ├── block_list.py        # список блоков сценария (drag&drop, подсветка)
        ├── block_editor.py      # инспектор: настройки блока/триггера/действий
        ├── trigger_list.py      # карточки триггеров с индикаторами
        └── color_picker.py      # пипетка + слайдеры HSV
```

Главный принцип: `core/` чистая логика, `ui/` только интерфейс. Движок работает в отдельном потоке (QThread) и общается с интерфейсом только через сигналы Qt поэтому окно не зависает во время работы бота.

Рабочие файлы программа хранит в `~/.botforge/`:
- `templates/` вырезанные картинки-образцы
- `sounds/` записанные звуковые образцы

## 📖 Микро-документация

### Формат проекта (.json)

Проект обычный JSON, его можно править руками или генерировать скриптом:

```json
{
  "name": "Мой бот",
  "loop": true,
  "loop_delay_ms": 50,
  "max_iterations": 0,
  "triggers": [
    {
      "id": "a1b2c3d4",
      "name": "Кнопка ОК",
      "type": "template",
      "region": [100, 200, 300, 150],
      "template_path": "C:/Users/.../.botforge/templates/a1b2c3d4.png",
      "template_threshold": 0.8
    }
  ],
  "blocks": [
    {
      "id": "e5f6a7b8",
      "type": "if_trigger",
      "trigger_id": "a1b2c3d4",
      "actions": [
        { "type": "click_on_trigger" },
        { "type": "wait", "ms": 500 },
        { "type": "key_press", "key": "enter" }
      ]
    }
  ]
}
```

Кратко по полям:

- `region` всегда `[x, y, ширина, высота]` в **физических** пикселях экрана
- `max_iterations: 0` бесконечный цикл; число остановиться после N проходов
- блоки `if_trigger` и `while_trigger` держат **список** `actions` они выполняются по очереди
- у `while_trigger` есть `invert: true` («пока триггера НЕТ»), `repeat_delay_ms` (пауза между повторами) и `max_repeats` (защита от вечного цикла)
- у `wait_trigger` есть `wait_timeout_ms` (0 = ждать бесконечно)

### Типы триггеров (поля)

| Типы         | Ключевые поля |
|--------------|---|
| `color_mask` | `hsv_lower`, `hsv_upper` диапазон HSV; `min_match_ratio` какая доля области должна быть этого цвета |
| `template`   | `template_path` путь к картинке; `template_threshold` порог похожести 0..1 (обычно 0.8) |
| `pixel`      | `pixel_x`, `pixel_y`, `pixel_rgb` ожидаемый цвет; `pixel_tolerance` допуск |
| `change`     | `min_match_ratio` доля изменившихся пикселей, чтобы считать движение есть |
| `sound`      | `sound_path` образец WAV; `sound_mode` `match` (похож на образец) или `level` (просто громко); пороги `sound_threshold` / `sound_level` |

### Типы действий

`click_xy`, `double_click`, `click_on_trigger`, `mouse_hold`, `drag`, `move_mouse`, `scroll`, `key_press`, `key_hold`, `text_type`, `wait`, `random_wait`, `beep`, `stop_bot`, `restart_scenario`

### Клавиши

В поле клавиши можно писать одиночные клавиши и комбинации через `+`:

```
enter   esc   f5   space   ctrl+s   ctrl+shift+a   win+d   alt+tab
```

### Горячие клавиши

| Клавиша | Действие |
|---|---|
| **F12** | Аварийная остановка бота (глобально, даже из игры) |
| **Del** | Удалить выбранный блок |
| Drag&Drop | Перестановка блоков в сценарии |
